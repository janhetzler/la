# Installation Guide

This guide covers the manual install in case the automated `./install.sh` fails, or if you want to understand each step before running it.

## Prerequisites

### Supported platforms

- **macOS** (Apple Silicon recommended) — full support, including meeting capture pipeline
- **Linux** (Ubuntu 22+, Fedora 38+) — partial support (manual audio loopback needed for meeting capture)
- **Windows 11** (with WSL2) — partial support

### Minimum hardware

- **RAM**: 12 GB minimum, 16 GB recommended
- **Disk**: 25 GB free
- **CPU**: Apple Silicon (M1 or newer) recommended; otherwise a recent 8+ core CPU

### Software to install manually first

| Component | macOS | Linux | Windows |
|---|---|---|---|
| Python 3.10+ | `brew install python@3.12` | `apt install python3.12 python3.12-venv` | python.org installer |
| Git | Xcode CLI tools | `apt install git` | git-scm.com |
| Homebrew | https://brew.sh | n/a | n/a |

The installer will help with: Docker, Ollama, Node.js, whisper.cpp.

## Quick install

```bash
git clone https://github.com/<your-username>/chief-of-staff.git
cd chief-of-staff
./install.sh        # macOS / Linux
# or
install.bat         # Windows
```

The installer runs **8 sequential steps**:

1. **Prerequisites** — verifies RAM, disk, Python, OS
2. **System tools** — checks Docker and Ollama presence
3. **AI models** — pulls Granite tiny-h, micro-h, embedding (~7 GB total)
4. **whisper.cpp** — clones, compiles, downloads Medium model (~1.5 GB)
5. **Python venv** — creates `.venv/` and installs `requirements.txt`
6. **Secrets** — interactively prompts for `GITHUB_TOKEN` and `TAVILY_API_KEY`, writes `.env`
7. **Docker stack** — starts Qdrant, Postgres, LiteLLM, Open WebUI
8. **End-to-end validation** — smoke test

Each step is independent — if one fails you can re-run `./install.sh` and it will skip what's already done.

## Manual install (step by step)

If you prefer to control each step yourself:

### 1. Container runtime

```bash
# macOS
brew install --cask orbstack    # recommended (or docker-desktop)

# Linux
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER   # logout/login required
```

Verify: `docker run --rm hello-world`

### 2. Ollama + Granite models

```bash
# Install Ollama
brew install ollama && brew services start ollama    # macOS
curl -fsSL https://ollama.com/install.sh | sh        # Linux

# Pull models (~7 GB)
ollama pull ibm/granite4:tiny-h
ollama pull ibm/granite4:micro-h
ollama pull granite-embedding:30m
```

### 3. whisper.cpp (for meeting transcription)

```bash
# macOS
brew install whisper-cpp

# Linux — build from source
git clone https://github.com/ggerganov/whisper.cpp ~/whisper.cpp
cd ~/whisper.cpp && make

# Download the Medium model (~1.5 GB)
cd <project-root>/models/whisper
curl -L -o ggml-medium.bin \
  https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-medium.bin
```

### 4. Python environment

```bash
cd <project-root>
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 5. Secrets

Create `.env` in the project root:

```bash
GITHUB_TOKEN=github_pat_...      # https://github.com/settings/tokens?type=beta
TAVILY_API_KEY=tvly-...          # https://tavily.com (free tier: 1000 req/month)
```

Both keys are optional — without them, the Code agent loses GitHub access and the Researcher loses web search, but the rest of the system still works.

### 6. MCP config

The `mcp/mcp.json` file is generated by step 06 of the installer. If you skipped the installer, run:

```bash
python -c "from installer.steps import step06_secrets; step06_secrets.run()"
```

Or copy `mcp/mcp.json.template` to `mcp/mcp.json` and replace `PROJECT_ROOT_PLACEHOLDER` with the absolute path to your project.

### 7. Start the stack

```bash
./start.sh
```

This brings up:
- Docker stack (Qdrant, Postgres, LiteLLM, Open WebUI) — ports 6333, 5432, 4000, 3000
- Specialists API (port 8001) — whisper.cpp wrapper, embeddings
- Agents API (port 8002) — the 6 LangGraph agents
- Library watcher — auto-ingests files dropped into `data/library/<category>/`

Open http://localhost:3000 and select `agent-chief-of-staff` in the model picker.

## Daily lifecycle

```bash
./start.sh     # ~30 seconds to bring everything up
./stop.sh      # clean shutdown
```

## Verification

After install, the smoke test should pass:

```bash
# Test 1: Ollama answers
curl http://localhost:11434/api/tags

# Test 2: LiteLLM lists models
curl http://localhost:4000/v1/models -H "Authorization: Bearer sk-cos-local-dev"

# Test 3: Agents API health
curl http://localhost:8002/health

# Test 4: Open WebUI loads
curl -I http://localhost:3000
```

All four should return 200 OK.
